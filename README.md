# Retail AI Marketing — PRD + Prototype

A complete Product Requirements Document paired with a working interactive prototype for an agentic AI retail marketing application.

Built as a PM capstone — covers the full product lifecycle from problem definition to technical specification to a shipped, runnable prototype.

We use the following tech stack:
- Google Gemini for AI-powered campaign generation and audience targeting
- TypeScript + React for the frontend prototype
- Tailwind CSS for styling

## 🚀 PM Artifacts Inside

### Product Requirements Document
* **Problem Statement & Market Sizing:** Why SMB retailers can't do personalized marketing at scale, and the opportunity this creates
* **User Personas:** Two primary personas — SMB store owner and marketing manager — with goals, pain points, and jobs-to-be-done
* **User Stories:** INVEST-formatted stories with clear acceptance criteria and definition of done
* **Success Metrics:** North Star metric, L1/L2 KPIs, guardrail metrics, and instrumentation plan
* **Prioritization:** RICE-scored feature backlog with rationale for MVP scope decisions
* **Go-to-Market Strategy:** Launch sequencing, pricing model, channel strategy, and onboarding flow
* **Roadmap:** Phased delivery — MVP → V1 → V2 with milestones and dependencies

### Working Prototype Features
* **Campaign Generator:** AI creates marketing copy, taglines, and creatives from product descriptions
* **Audience Segmentation:** Segments customers by purchase behavior, location, and demographics
* **Channel Recommender:** Suggests best channels (email, WhatsApp, social) per audience segment
* **Performance Estimator:** Predicts reach and estimated conversion rate before launch

## Setup

```bash
git clone https://github.com/yatinbhalla/Retail-AI-Marketing-PRD-Prototype.git
cd Retail-AI-Marketing-PRD-Prototype
npm install
echo "GEMINI_API_KEY=your_key_here" > .env.local
npm run dev
```
